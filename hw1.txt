1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?

Для этого необхдимо выполнить следующие шаги:

    a) Установить Git на свой компьютер.
    b) Авторизоваться на GitHub.com.
    c) Перейти на страницу репозитория, который необходимо скопировать.
    d) Скопировать URL репозитория (HTTPS или SSH).
    e) Открыть терминал или командную строку на своем компьютере.
    f) Перейти в папку, куда нужно сохранить копию репозитория.
    g) В терминале или командной строке выполнить команду git clone с указанием скопированного URL репозитория. Например, как было сделано на занятии "git clone git@github.com:AnnaDDD/240123-m.git".

2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

Для этого необхдимо выполнить следующие шаги:

    a) Открыть терминал или командную строку на своем компьютере.
    b) Перейти в каталог, где необходимо создать репозиторий. Например, если это нужно сделать в папке "homework", то  следует выполнить команду:cd homework или открыть гитбаш сразу в необходимой папке.
    c) Инициализировать Git репозиторий в этом каталоге командой git init. 
    d) Добавить файлы в Git репозиторий командой git add (например, чтобы добавить все файлы в текущей папке, выполнить команду "git add ." или "git add file_name.txt" если нужно добавить конкретный файл или несколько)
    e) Сохранить изменения в локальном репозитории командой git commit с сообщением о коммите: git commit -m "first commit"
    f) Создать репозиторий на GitHub.com.
    g) Связать локальный репозиторий с репозиторием на GitHub.com командой git remote add. Например: git remote add origin git@github.com:AnnaDDD/Git_Prof_HW.git
    h) Отправить изменения в репозиторий на GitHub.com командой git push. Например: git push --set-upstream origin master


3. Как получить на локальный компьютер изменения с github.com

Для этого необхдимо выполнить следующие шаги:

    a) Открыть терминал или командную строку на своем компьютере.
    b) Перейти в каталог локального репозитория, где вы хотите получить изменения. Например, если вы хотите получить изменения в папке "projects", то выполните команду: cd projects
    c) Выполнить команду git pull с указанием названия удаленного репозитория. Например:git pull origin master (если синхронизируем изменения между локальным репозиторием и репозиторием на GitHub.com для ветки master, если нужно получить изменения из другой ветки, то нужно указать ее название вместо master). После выполнения команды git pull, Git автоматически скачает все изменения из удаленного репозитория на GitHub.com и объединит их с текущей локальной веткой.


4. Какую информацию нам дают команды : git status, git log, git remote -v ?

    a) git status — показывает текущее состояние локального репозитория, включая изменения, которые были сделаны в файлах, но еще не добавлены в коммит. Так же git status показывает ветку, на которой находится локальный репозиторий, и наличие каких-либо изменений в удаленном репозитории.

    b) git log — показывает список всех коммитов в текущей ветке локального репозитория, начиная с самого нового. Для каждого коммита показываются дата и время коммита, автор коммита, и сообщение о коммите, которое было добавлено при его создании.

    c) git remote -v — показывает список всех удаленных репозиториев, связанных с текущим локальным репозиторием, а также URL для чтения и записи для каждого удаленного репозитория. Кроме того, эта команда показывает название короткого имени каждого удаленного репозитория, которое может использоваться в других командах Git для обращения к нему.
